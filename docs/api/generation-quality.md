# 生成比例与画质约定

客户端同时发送新旧字段以兼容历史版本：

- 新字段：`aspectRatio`、`quality`
- 兼容字段：`ratio`、`resolution`

服务端优先读取新字段，并通过 `lib/generation-quality.ts` 统一解析。任务参数保存请求画质、`actualQuality`、`qualityFallbacks` 与 `finalSize`。

## 图片尺寸

| 比例 | 1K | 2K |
|---|---:|---:|
| 1:1 | 1024×1024 | 2048×2048 |
| 4:3 | 1184×864 | 2368×1728 |
| 3:4 | 864×1184 | 1728×2368 |
| 16:9 | 1344×768 | 2688×1536 |
| 9:16 | 768×1344 | 1536×2688 |

## 降级规则

- 图片：2K → 1K。
- 视频：1080p → 720p → 480p，并过滤模型不支持的档位。
- 仅匹配 resolution、size、dimensions、width/height、pixel、OOM/CUDA 等画质类错误。
- API Key、鉴权、余额、支付、限流、审核或安全策略错误绝不自动重试。
- 上游已返回任务 ID 后，只有任务明确失败且错误属于画质类时才创建下一档任务；比例始终保持不变。
