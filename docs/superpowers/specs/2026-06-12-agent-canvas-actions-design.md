# Genora Agent 画布操作设计

## 目标

让 Agent 在理解用户自然语言后，可以安全地建议或执行画布操作，包括添加节点、填写节点提示词、连接节点和引用对话附件。当前阶段只规划，不修改 Agent 运行代码。

## 当前问题

- 前端只把画布摘要、附件名称和对话历史发送给 Agent。
- 后端只返回自然语言 `text`，没有可执行的结构化操作。
- 前端收到回复后只追加聊天消息，不会调用现有的 `addNode`、`update` 或连线逻辑。
- 模型回复格式没有 Schema 约束，容易出现答非所问、遗漏画布上下文或无法解析的内容。

## 推荐方案：结构化操作协议

Agent 返回自然语言说明和独立的操作列表：

```json
{
  "reply": "我准备了一个视频节点，并填入了提示词。",
  "actions": [
    {
      "type": "addNode",
      "clientActionId": "action-1",
      "kind": "video",
      "position": { "mode": "viewport-center" },
      "data": {
        "prompt": "一只猫在草地上奔跑，柔和自然光，电影镜头"
      }
    }
  ]
}
```

后端负责要求模型输出符合 JSON Schema 的数据；前端只执行白名单操作，不执行模型生成的任意代码。

## 首期操作白名单

- `addNode`：添加文本、图像或视频生成节点。
- `updateNodePrompt`：填写或追加节点提示词。
- `connectNodes`：连接两个已存在的节点。
- `attachConversationImage`：把用户在 Agent 中上传的图片作为新素材节点，或作为视频节点参考图。
- `focusNode`：定位并选中目标节点。

首期不支持 Agent 删除节点、修改项目名称、保存项目或直接访问本地文件。

## 执行权限

### 可自动执行

- 添加空节点。
- 填写新节点的提示词。
- 聚焦 Agent 刚创建的节点。
- 在同一批 Agent 操作中新建节点之间建立连接。

### 必须确认

- 开始图片或视频生成。
- 覆盖用户已经填写的提示词。
- 使用上传附件发起远端生成。
- 删除节点、断开连线或批量修改画布。
- 任何可能产生费用或不可逆结果的操作。

确认界面应展示 Agent 即将执行的动作摘要，而不是直接弹出原始 JSON。

## 数据流

1. 前端生成稳定的画布上下文，包括节点 ID、类型、提示词摘要、任务状态和连线关系。
2. 后端使用固定系统提示词和 JSON Schema 请求 Agnes 文本模型。
3. 后端解析并校验响应；无效响应最多进行一次格式修复请求。
4. 前端再次校验操作类型、字段长度、节点 ID 和权限等级。
5. 自动操作立即执行；高风险操作进入确认队列。
6. 每个操作返回成功或失败结果，Agent 对话中显示实际执行结果。

## 回复异常处理

- 系统提示词明确区分 `reply` 与 `actions`，禁止在 Markdown 代码块外输出额外 JSON。
- 使用 Schema 限制节点类型、操作类型、提示词长度和必填字段。
- 解析失败时保留原始文本作为普通回复，不执行任何画布操作。
- 操作引用不存在的节点时跳过该操作，并返回可读错误。
- 同一个 `clientActionId` 只执行一次，避免重试产生重复节点。
- 模型不能判断目标节点时，先询问用户，不猜测节点 ID。

## 前端边界

建议新增独立的 `applyAgentActions` 层，负责校验和调用现有画布函数。Agent 面板不直接操作 React Flow 状态，避免聊天逻辑与画布状态耦合。

操作执行后应使用现有未保存标记与项目自动保存机制，不建立第二套保存流程。

## 测试重点

- 合法 `addNode` 能创建正确节点并填写提示词。
- 非白名单操作被拒绝。
- 重复 `clientActionId` 不会创建重复节点。
- 不存在的节点 ID 不会破坏画布。
- 生成操作必须经过确认。
- 模型返回非 JSON 时仍能显示普通回复。
- 多操作中单个失败不会阻止其余安全操作，并能返回逐项结果。

## 分阶段实施

1. 固定 Agent 回复 Schema，仅支持 `addNode` 和 `updateNodePrompt`。
2. 增加操作预览与生成确认。
3. 支持连接节点和使用 Agent 附件。
4. 增加操作历史、撤回和更精细的权限控制。
