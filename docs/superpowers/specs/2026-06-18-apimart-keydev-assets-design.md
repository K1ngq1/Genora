# APIMart KeyDev 与画布资产持久化设计

## 目标

- 修复 APIMart 任务在取得远端任务 ID 前失败却只显示 `UNKNOWN_ERROR` 的问题。
- 使用 `APIMART_KEY_DEV` 接入当前公开价格最低、适合画布测试的图片与视频模型，并各执行一次真实生成验证。
- 将节点上传的图片持久化到服务端，确保刷新页面或重启服务后仍可作为生成参考图使用。

## 模型选择

- 图片：`gpt-image-2-official`，默认 `1:1`、`1k`、`low`。APIMart 公开价格页中，该组合低于现有 `gpt-image-2` 1K 档。
- 视频：`grok-imagine-1.5-video-apimart`，默认 `16:9`、`720p`、`6s`。它是当前公开价格目录中的最低价视频生成档。
- 仅这两个测试模型读取 `APIMART_KEY_DEV`；已有模型继续读取图片或视频专用 Key，避免改变现有生产配置。

## APIMart 适配

- Provider 请求层按模型选择 Key，不向日志、数据库或前端返回密钥。
- 为两个模型生成官方文档要求的请求体：图片增加 `quality: low`；视频使用 `size`、`quality`、`duration`。
- 保留统一 `/v1/tasks/{task_id}` 轮询，并兼容 `submitted`、`queued`、`processing`、`completed`、`failed`。
- 上游失败时保存错误代码和经过截断、去敏后的详情，避免再次只出现 `UNKNOWN_ERROR`。

## 资产持久化

- Prisma 新增 `Asset` 表，保存所属项目、稳定文件路径、原始文件名、MIME、字节数和创建时间。
- 文件二进制保存在服务端 `storage/uploads`，数据库保存资产记录，避免将大图片 BLOB 塞入 SQLite。
- 上传接口要求 `projectId`，创建资产后返回 `/api/assets/{id}` 稳定 URL。
- 图片素材、图片节点参考图、视频首尾帧均先上传服务端，再写入节点数据；不再把 `blob:` URL 保存进项目 JSON。
- 资产接口只返回文件内容，不提供前端数据库列表或管理界面。
- 本期不自动删除孤立资产，避免误删仍被历史项目或任务引用的文件。

## 数据流

1. 用户在节点选择本地图片。
2. 前端立即上传文件与当前 `projectId`。
3. 服务端保存文件并创建 `Asset` 记录，返回稳定 URL。
4. 节点保存稳定 URL；项目自动保存后，刷新和重启均可恢复。
5. 生成时服务端读取稳定资产并上传到 APIMart，再提交模型任务。

## 验证

- 自动测试覆盖模型目录、Key 路由、两个请求体、错误详情、资产上传和节点不保存 `blob:` URL。
- 执行 Prisma 本地迁移、`npm run check` 和 `npm run build`。
- 使用 `APIMART_KEY_DEV`：生成一张低成本图片和一个 6 秒低成本视频，轮询至完成并确认结果已下载到本地。
- 重启开发服务器后重新读取项目，确认节点参考图片 URL 可访问且生成请求仍包含参考图片。

## 边界

- 不部署生产环境，不修改 Supabase，不执行真实数据批量清理。
- 真实付费调用仅限图片、视频各一次；失败时先诊断，不自动重复付费提交。
